### HTTP 주요 메소드

**`GET` :**  특정 리소스의 표시 요청 / 오직 데이터를 받기만 함

**`HEAD` :** `GET` 메소드의 요청과 동일한 응답을 요구하지만 본문 포함 X

**`POST` :** 특정 리소스에 엔티티 제출

**`PUT`** : 타겟 리소스의 모든 현재 표시를 요청 payload 로 바꿈

…

### HTTP 주요 Status code

> 특정 HTTP 요청이 성공적으로 완료되었는지 알림
> 
- **정보 응답**
    
    요청을 받았으며 프로세스를 계속 진행한다는 의미
    
    - `100 Continue` : 현재까지의 요청이 괜찮으니, 클라이언트는 본문(body)을 계속 전송해도 좋음
- **성공 응답**
    
    요청이 성공적으로 수신,이해,처리됨
    
    - `200 OK`: 요청 성공적으로 완료됨
    - `201 Created`: 요청 성공했으며, 결과로 새 리소스 생성됨
    - `204 No Content`: 요청 성공했으나, 응답으로 보낼 콘텐츠 X
- **리다이렉션 메시지**
    
    요청을 완료하기 위해 추가적 조치 필요
    
    - `301 Moved Permanently`: 요청한 리소스의 URL이 영구적으로 변경 / 새 URL이 응답에 포함됨
    - `302 Found`: 요청한 리소스가 일시적으로 다른 URL에 있음. (임시 리다이렉션)
    - `304 Not Modified`: 리소스가 수정되지 않았음. 클라이언트는 캐시된 버전을 사용해야 함.
- **클라이언트 에러 응답**
    
    클라이언트 측의 오류로 요청 완료 불가
    
    - `400 Bad Request`: 서버가 클라이언트의 요청 이해할 수 X (예: 잘못된 요청 구문)
    - `401 Unauthorized`: **인증**이 필요한 리소스임
    - `403 Forbidden`: 리소스에 대한 **접근 권한**이 없음
        
        (인증은 되었으나 권한 부족)
        
    - `404 Not Found` : 서버가 요청한 리소스를 찾을 수 없음
- **서버 에러 응답**
    
    서버측의 문제로 유효한 요청 처리 불가
    
    - `500 Internal Server Error` : 서버 내부에서 예상치 못한 오류 발생
    - `502 Bad Gateway` : 게이트웨이나 프록시 서버가 상위 서버로부터 잘못된 응답을 받음
    - `503 Service Unavailable` : 서버가 일시적인 과부하 또는 점검으로 요청 처리 준비 안 됨.
