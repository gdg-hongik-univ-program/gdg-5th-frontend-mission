# HTML의 기초

## 프론트엔드 개발자가 하는 일
* 애플리케이션을 사용하는 사람이 애플리케이션과 소통하기 위한 유저 인터페이스를 사용하기 좋게 구현

## 프로그래밍 학습 방법
* 그냥 의식적인 연습을 꾸준히 반복해라, 기본기가 중요하다

---

## 요소 (Element)
기본적으로 HTML 문서는 요소들의 집합으로 이루어집니다. `<html>` 요소가 `<body>` 요소를 포함하는 **Nested Element** 구조로 표현합니다.

## 어트리뷰트 (Attribute)
요소의 성질, 특징을 정의하는 명세로 요소에 추가적인 정보를 제공합니다. (항상 **시작 태그**에 위치)

---

## 시맨틱 태그 (Semantic Tags)
시맨틱 태그는 단순히 글자를 꾸미는 태그와 달리, 브라우저, 검색엔진, 개발자에게 콘텐츠의 **의미와 구조를 명확하게 전달**합니다.
이렇게 의미를 명확히 하면 코드 가독성과 유지보수성이 높아지고, 검색엔진이 웹페이지의 중요도를 파악해 더 효과적으로 인덱싱할 수 있게 됩니다.

---

## _**<head>**_ 태그 내부의 주요 요소
* **`<title>`**: 문서의 제목(브라우저 탭에 표시)을 정의합니다.
* **`<style>`**: 태그의 배경과 글자 색상 등 CSS style 정보를 정의합니다.
* **`<link>`**: 외부 리소스와의 연계 정보를 정의합니다. (주로 외부 CSS 파일 연동에 사용)
    * 예: `<link rel="stylesheet" href="style.css">`
* **`<script>`**: Client-side JavaScript를 정의합니다. `src` 속성으로 외부 JavaScript 파일을 로드할 수 있습니다.
* **`<meta>`**: 웹페이지의 추가 정보(메타데이터)를 정의하며 이 정보는 브라우저나 검색엔진이 사용합니다.
    * 예: `<meta charset="UTF-8">`

## _**<body>**_ 태그 내부의 주요 요소

### 1. 제목 태그 (Headings)
* **`<h1>` ~ `<h6>`**: 문서의 제목(headings)을 나타냅니다. 시맨틱 웹의 의미를 살려 제목 이외에는 사용하지 않는 것이 좋습니다.

### 2. 글자 형태 태그
* **`<b>`**: 단순히 bold체를 지정합니다.
* **`<strong>`**: bold체를 지정하고, 의미론적 중요성의 의미를 가집니다.
* **`<i>`**: 단순히 Italic체를 지정합니다.
* **`<em>`**: Italic체를 지정하고, 의미론적 강조의 의미를 가집니다.
* **`<small>`**: 기존 글씨 크기보다 작은 텍스트를 지정합니다.
* **`<mark>`**: 텍스트에 하이라이트 표시를 합니다.
* **`<del>`**: 텍스트는 남기지만 가운데 줄을 그어 삭제 표시를 합니다.
* **`<ins>`**: 밑줄을 그어 추가된 텍스트를 표시합니다.
* **`<sub>` / `<sup>`**: 각각 하단에 쓰인 텍스트 그리고 상단에 쓰인 텍스트를 지정합니다.

### 3. 본문 태그
* **`<p>`**: 단락을 지정합니다.
* **`<br>`**: 줄 바꿈을 지정합니다. (빈 요소로 종료 태그가 없음)
    * 연속적인 공백은 `&nbsp;`을 이용해 삽입합니다.
* **`<pre>`**: 내부에 작성된 요소는 그대로 브라우저에 표시됩니다.
* **`<hr>`**: 수평줄을 삽입하여 주제 변경 등을 나타냅니다.
* **`<q>`**: 짧은 인용문을 나타내며 큰따옴표로 요소를 감쌉니다.
* **`<blockquote>`**: 블록 단위의 긴 인용문을 지정합니다. 브라우저는 보통 요소를 들여쓰기해서 표현하며 CSS로 스타일 적용이 가능합니다.

## 하이퍼링크
`<a>` 태그(Anchor)를 이용하여 한 텍스트에서 다른 텍스트로 건너뛰어 읽을 수 있도록 하는 기능입니다.

### href 속성
* **절대 경로**: `http://www.google.com`처럼 완전한 URL 주소입니다.
* **상대 경로**: `~/.html`처럼 현재 파일을 기준으로 경로를 지정합니다.
* **Fragment Identifier**: `#top`처럼 페이지 내부의 특정 ID로 이동합니다.
* **메일**: `mailto:email@example.com`
* **Script**: `javascript:alert('Hello');`

### target 속성
* **`_self`**: 링크를 클릭했을 때 현재 윈도우에서 오픈 (기본값)
* **`_blank`**: 링크를 클릭했을 때 새 탭에서 오픈 (보안 취약점 때문에 `rel="noopener noreferrer"` 추가 권장)

## 목록 및 테이블

### 목록 (List)
* **`<ul>`** : 순서 없는 목록 (앞에 • 같은 점으로 표현)
* **`<ol>`** : 순서 있는 목록 (앞에 1., 2. 같은 숫자로 표현)
    * `type` 속성: 순서를 나타내는 문자 지정 (예: `A`, `a`, `I`, `i`)
    * `start` 속성: 리스트의 시작값을 지정 (예: `5`부터 시작)
    * `reversed` 속성: 리스트의 순서값을 역으로 표시
* **`<li>`** : `<ul>`와 `<ol>` 내부에서 실제 목록 항목을 나타냅니다.

### 테이블 (Table)
* **`<table>`**: 표를 감싸는 태그
* **`<tr>`** : 표 내부의 행
* **`<th>`** : 행 내부의 제목 셀 (보통 굵게, 중앙 정렬)
* **`<td>`** : 행 내부의 일반 셀

## 미디어

### 이미지 `<img>`
* **`<img>`** 태그를 이용하며, 빈 요소입니다.
* **`src`** (source): 이미지 파일 경로
* **`alt`** (alternative): 이미지가 로드되지 않을 때 대신 표시될 텍스트
* **`width`**: 이미지 너비
* **`height`**: 이미지 높이

### 오디오 `<audio>`
* **`<audio>`** 태그를 이용합니다.
* **`src`**: 음악 파일 경로
* **`preload`**: 재생 전에 모두 불러올 것인지 지정
* **`autoplay`**: 자동 재생할 것인지 지정
* **`loop`**: 반복 재생할 것인지 지정
* **`controls`**: 음악 재생 도구(컨트롤러)를 표시할 것인지 지정
* **`(참고)`**: 브라우저별로 지원하는 파일 형식이 다를 때, `src` 태그로 해결 가능

### 비디오 `<video>`
* **`<video>`** 태그를 이용하며, 속성은 `<audio>`와 유사합니다.
* **`poster`**: 동영상 준비 중(로딩 중)에 표시될 이미지 파일 경로
* **`width` / `height`**: 비디오 너비/높이 지정
* **`(참고)`** : 오디오와 마찬가지로 파일 형식 차이를 `src` 태그로 해결

## 입력 양식

* **`<form>`**: 사용자 입력을 받아 `action` 속성에 지정된 URL로 데이터를 전송합니다.
    * **`method="GET"`**: 데이터를 URL에 `?key=value` 형태로 붙여서 공개적으로 보냅니다. (주로 조회용)
    * **`method="POST"`**: 데이터를 요청 본문(Body)에 숨겨서 보냅니다. (주로 생성/제출용)

* **`<input>`**: 사용자로부터 데이터를 입력받는 핵심 태그입니다.
    * **`type`** 속성에 따라 텍스트(`text`), 비밀번호(`password`), 버튼(`button`), 제출(`submit`) 등 다양한 입력 방식을 정의합니다.
    * 서버에는 `name` 속성값을 키(key)로, `value` 속성값을 값(value)으로 하여 `key=value` 형태로 전송됩니다.

* **`<select>` / `<option>`**: 드롭다운 목록을 만듭니다. `<select>`가 전체를 감싸고, 여러`<option>`항목으로 구성되어 있습니다.

* **`<textarea>`**: 여러 줄의 글자를 입력할 때 사용합니다. (메모장 느낌)

* **`<button>`**: 클릭할 수 있는 버튼을 생성합니다. (`type` 속성: `button`, `submit`, `reset` 지정을 권장합니다.)

* **`<fieldset>` / `<legend>`**: `<fieldset>`은 입력 양식들을 그룹화할 때 사용하고, `<legend>`는 해당 그룹의 제목을 정의합니다.

## 웹페이지 레이아웃
주로 `<div>`를 이용해 레이아웃을 구성했으나, HTML5부터는 각 구역의 의미를 명확하게 정의하는 시맨틱 태그 사용을 권장하고 있습니다.

* **`<header>`**: 페이지 상단, 머리글 영역
* **`<nav>`**: 내비게이션(메뉴) 링크 영역
* **`<section>`**: 문서의 주요 구역을 나눌 때 사용
* **`<article>`**: 블로그 글, 뉴스 기사처럼 독립적으로 배포 가능한 내용
* **`<aside>`**: 본문 옆의 사이드바 영역
* **`<footer>`**: 페이지 하단, 바닥글 영역
* **`<div>`**: 위 태그들로 의미를 정의할 수 없을 때 사용하는 순수 컨테이너
