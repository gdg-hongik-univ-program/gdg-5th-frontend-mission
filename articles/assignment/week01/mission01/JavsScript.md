1. 데이터 타입
- number : 숫자를 나타내는 데 사용하는 단 하나의 숫자 타입으로 64비트 배정밀도 부동소수점형
ex) var n = 3;
- string : 텍스트 데이터를 나타내는 데 사용하는 문자열 타입으로 변경 불가능
ex) var s = "string";
- boolean : 논리적 참과 거짓을 나타네는 값으로 true와 false만 존재
ex) var b = true;
- undefined : 선언 이후 값을 할당하지 않은 변수가 가지는 값
ex) var u;
- null : 의도적으로 변수에 값이 없다는 것을 명시할 때 사용하는 타입
ex) var n = null;
- symbol : Symbol 함수를 통해서 유일한 객체의 키를 만들기 위해 사용하는 타입
ex) var key = Symbol('key');
- object : 객체는 데이터를 의미하는 프로퍼티와 동작을 의미하는 메소드를 포함하는 독립적 주체

2. 변수
- 변수 : 프로그램에서 사용되는 데이터를 일정 기간 동안 기억하여 필요한 때에 다시 사용하기 위해 데이터에 고유의 이름인 식별자를 명시한 것
- 선언 방법 : var, let, const 키워드로 선언 가능
- 동적 타이핑 : 변수의 타입 저장 없이 값이 할당되는 과정에서 값의 타입에 의해 타입이 자동으로 결정
- 변수 호이스팅 : 선언 단게 -> 초기화 단계 -> 할당 단계

3. 연산자
- 연산자 : 하나 이상의 표현식을 대상으로 연산을 수행해 하나의 값을 만드는 것
- 산술 연산자 : 2개 피연산자를 대상으로 연산해 숫자 타입의 값을 만듦
ex) +, -, *, /, %, ++, --
- 문자열 연결 연산자 : 피연산자 중 하나 이상이 문자열인 경우 작동
ex) '1' + '2'
- 할당 연산자 : 우항에 있는 피연산자의 평가 결과를 좌항에 있는 변수에 할당
ex) =, +=, -=, *= /=, %=
- 비교 연산자 : 좌항과 우항의 피연산자를 비교해 불리언 값을 반환
ex) ==, ===, !=, !==, >, <, >=, <=
- 삼항 조건 연산자 : 조건식의 ㅍㅇ가 결과에 따라 반환할 값을 결정
ex) ?
- 논리 연산자 : 우항과 좌항의 피연산자를 논리 연산해 불리언 값 반환
ex) ||, &&, !
- 쉼표 연산자 : 왼쪽 피연산자부터 차례대로 평가
ex) ,
- 그룹 연산자 : 그룹 내의 표현식을 최우선으로 평가
ex) (, )
- typeof 연산자 : 피연산자의 데이터 타입을 문자열로 반환

4. 조건문
- if else문 : 주어진 조건식의 평가 결과에 따라 실행할 코드 블록을 결정
ex) if (n > 0) {
    console.log('pos');
    }
- switch문 : 표현식을 평가해 그 값과 일치하는 표현식을 갖는 case문으로 이동
ex) switch (month) {
        case 1:
            monthName = 'January';
        case 2:
            monthName = 'February';
        ...
    }

5. 반복문
- for문 : 조건식이 거짓으로 판별될 때까지 코드 블록을 반복 실행
ex) for (var i = 0; i < 2; i++) {
        console.log(i);
    }
- while문 : 조건식의 평가 결과가 참이면 코드 블록을 게속해서 반복 실행
ex) while (count < 3) {
        console.log(count);
        count++;
    }
- do while문 : 코드 블록을 실행하고 조건식을 평가
ex) count = 0;
    do {
        console.log(count);
        count++;
    } while (count < 3);

6. break문, continue문
- break문 : 레이블문, 반복문, switch문의 코드 블록을 탈출
ex) if (true) {
        break;
    }
- continue문 : 반복문의 코드 블록 실행을 현 지점에서 중단하고 반복문의 증감식으로 이동

7. 타입 변환
- 명시적 타입 변환 : 개발자에 의도적으로 값의 타입을 변환
- 암묵적 타입 변환 : 개발자의 의도와 상관 없이 타입이 자동으로 변환

8. 객체
- 객체 : 키와 값으로 구성된 프로퍼티들의 집합
- 프로퍼티 : 프로퍼티 키와 프로퍼티 값으로 구성된 객체의 데이터
- 메소드 : 함수인 프로퍼티로 객체에 제한되어 있음

9. pass-by-reference와 pass-by-value
- pass-by-reference : object 타입은 연산이 실제 값이 아니라 참조값으로 처리됨
- pass-by-value : 원시 타입은 실제 값이 복사되어 전달됨

10. 객체의 변경 불가성
- 변경 불가성 : 객체가 생성된 이후 그 상태를 변경할 수 없는 디자인 패턴
- 원시 타입 : 변경 불가능
- 객체 타입 : 변경 가능
- 불변 데이터 패턴 : 의도하지 않은 객체의 변경이 발생하는 원인을 막기 위해 객체의 방어적 복사를 통해 새로운 객체를 생성한 후 변경
ex) Object.assign(), Object.freeze()

11. 함수
- 함수 : 어떤 작업을 수행하기 위해 필요한 문들의 집합을 정의한 코드 블록
- 함수 선언문 : 함수명, 매개변수 목록, 함수 몸체를 사용해서 함수를 선언 가능
- 함수 호이스팅 : 함수 선언, 초기화 , 할당이 한번에 이루어짐

12. 매개변수
- 매개변수 : 함수의 작업 실행을 위해 추가적인 정보가 필요한 경우 지정
- call-by-value : 원시 타입 인수는 값에 의한 호출로 동작
    - 함수 호출 시 매개변수에 값을 복사해 함수로 전달
- call-by-reference : 객체 타입 인수는 참조에 의한 호출로 동작
    - 함수 호출 시 매개변수에 값이 복사되지 않고 참조값이 매개변수에 저장되어 함수로 전달되는 방식

13. 함수 객체의 프로퍼티
- arguments : 함수 호출 시 전달된 인수들의 정보를 담고 있는 유사 배열 객체
- caller : 자신을 호출한 함수
- length ; 함수 정의 시 작성된 매개변수 개수
- name : 함수명
- __proto__ 접근자 : 프로토타입 객체에 접근
- prototype : 함수 객체만이 소유하는 프로퍼티

14. 타입 체크
- typeof : 피연산자의 데이터 타입을 문자열로 반환
- Object.ptotype.toString : 객체를 나타내는 문자열을 반환
- instanceof : 객체의 상속 관계까지 체크 가능

15. 프로토타입
- 프로토타입 : 부모 역할을 담당하는 객체로 공유 프로퍼티를 제공하기 위해 사용
- [[Prototype]] : 함수를 포함한 모든 객체가 가지고 있는 인터널 슬롯
- prototype : 함수 객체만 가지고 있는 프로퍼티
- prototype chain : 객체의 프로퍼티를 참조하는 경우, 해당 객체에 프로퍼티가 없는 경우 동작

16. 스코프
- 전역 스코프 : 코드 어디에서든지 참조 가능
- 지역 스코프 : 함수 코드 블록이 만든 스코프로 함수 자신과 하위 함수에서만 참조 가능
- 전역 변수 : 전역에서 선언된 변수로 어디에든 참조 가능
- 지역 변수 : 지역 내에서 선언된 변수로 그 지역과 그 지역의 하부 지역에서만 참조 가능
- 함수 레벨 스코프 : 함수 코드 블록 내에서 선언된 변수는 함수 코드 블록 내에서만 유효
