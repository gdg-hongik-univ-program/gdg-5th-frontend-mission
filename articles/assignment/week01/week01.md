## 1번째 미션 : HTML, CSS, JavaScript 학습하기 

### 왜 이 미션이 중요할까요?
프론트엔드 개발자가 되기 위해서는 웹 페이지가 어떻게 구성되어 있는 지를 먼저 알 필요가 있어요. 
그 중심에 있는건 HTML과 CSS라는 언어예요.

HTML은 웹 페이지의 뼈대로, 어떤 요소로 구성되어 있는 지를 정의할 수 있어요. 
CSS는 웹 페이지를 꾸밀 수 있으며, 
JavaScript는 웹 페이지가 어떻게 동작하고 상호작용하는지를 정의해요.

이 기본기를 탄탄히 다져 놓으면 React를 배울 때도 훨씬 수월할 거예요!

[프론트엔드 참고 사이트](https://poiemaweb.com/)

모든 내용을 외우기보다 '이런 기능이 있구나' 정도로 익히는 것이 중요합니다.

하지만 TailwindCSS는 **유틸리티 우선** 방식으로, HTML 태그 안에서 바로 스타일을 적용할 수 있게 해줘요. 훨씬 빠르고 효율적으로 일관된 디자인을 만들 수 있게 도와주는 현대적인 스타일링 방식입니다.

TailwindCSS의 장점을 살펴보고 어떤 방식으로 사용하는지 학습해보아요.

## 2번째 미션 : pnpm vs npm 패키지 매니저 차이 알기  

### 왜 이 미션이 중요할까요?
우리가 웹 개발을 할 때 다른 사람들이 만들어 놓은 수많은 유용한 도구들(패키지, 라이브러리)을 사용하게 되는데, 
이 패키지들을 관리해주는 도구가 바로 패키지 매니저예요. 
가장 흔히 쓰이는 npm부터, 최근 성능과 효율성으로 주목받는 pnpm까지, 이들의 차이점을 알아두면 프로젝트의 환경 설정 시 현명한 선택을 할 수 있답니다. 
특히 pnpm은 디스크 공간을 효율적으로 사용하고 설치 속도가 빨라요. 

[패키지 매니저 참고 사이트](https://javierju.github.io/react/npm-vs-pnpm/)

[패키지 매니저 참고 사이트](https://mugglim.tistory.com/46)

### 미션 내용
npm과 pnpm의 차이점과 작동 방식(특히 node_modules 폴더의 구조나 심볼릭 링크 활용 방식)을 공부하고 정리해 봐요. 
둘 중 어떤 패키지 매니저를 사용할지 자유롭게 선택하여 앞으로의 미션에 적용해 봅시다.

[ ] npm과 pnpm의 주요 차이점 공부하고 정리해요.

## 3번째 미션 : React 프로젝트 생성하기 

### 왜 이 미션이 중요할까요?
드디어 우리가 만들 쇼핑몰의 메인 도구인 React를 만나볼 시간입니다. 
React는 사용자 인터페이스(UI)를 구축하는 데 가장 널리 사용되는 JavaScript 라이브러리예요. 
React를 효율적으로 사용하는 핵심 비결은 바로 컴포넌트(Component) 개념입니다.

**컴포넌트란?** 재사용이 가능하고 독립된 작은 UI 조각이에요. 
레고 블록처럼 각각의 컴포넌트를 만들고 조합하여 하나의 큰 애플리케이션 화면을 만들어 나가는 것이죠. 
화면을 컴포넌트로 잘게 나누면 코드를 이해하기 쉽고, 유지보수가 편리하며, 재사용성이 극대화됩니다.

### 미션 내용
**Part1. 프로젝트 생성하기**

[ ] Node.js를 설치하고 Vite를 사용하여 React 프로젝트를 생성합니다.(터미널 명령어 : pnpm create vite@latest 또는 선택한 패키지 매니저의 해당 명령어 사용)

**Part2. 쇼핑몰 홈화면 UI 구현**

[ ] 생성한 React 프로젝트 내에서 쇼핑몰 홈화면을 구현해 봐요.

[ ] TailwindCSS를 사용하여 스타일링을 완성해요.

**필수 구현 요구사항**
1. 컴포넌트 분리(최소 3개 이상) : 메인 화면을 최소한 3개의 컴포넌트로 나누어 구조화해야 합니다.

2. GDG 로고 이미지 삽입: GDG 로고를 표시할 두 가지 위치를 구현합니다.
- img 태그를 사용하여 로고를 삽입해요.
- 왼쪽 상단에 하나를 위치 시키고, 화면 정중앙에 희미하게 로고 하나를 위치 시킵니다.

3. Navbar 기능 구현: Navbar 내에 있는 버튼/div를 클릭했을 때 콘솔창에 해당 문구가 출력되도록 구현합니다.

- 카테고리 필터링 클릭 시 -> "카테고리 필터링 클릭" 출력

- 가격 범위 필터링 클릭 시 -> "가격 범위 필터링 클릭" 출력

- 상품 정렬 클릭 시 -> "상품 정렬 클릭" 출력

- 관리자 버튼 클릭 시 -> "관리자 페이지 클릭" 출력

4. Content 부분 구현: input 태그를 활용하여 검색바 UI를 구현하고, 검색 버튼 클릭 시 콘솔창에 검색한 대상이 출력되도록 합니다. (예: 원피스 검색 시 -> "원피스" 출력)
