1. HTML, CSS, JavaScript 학습하기
: 모든 웹 개발은 이 세 가지 기본 요소에서 시작한다. HTML은 웹페이지의 '뼈대(구조)'를 잡고, CSS는 '피부(디자인)'를 입히며, JavaScript는 '신경(상호작용)'을 담당한다. React로 만든 컴포넌트들도 결국 이 세 가지 요소의 조합으로 변환되어 브라우저에 표시된다.

2. pnpm vs npm 패키지 매니저 차이 알기
: npm과 pnpm은 모두 react-router-dom 같은 외부 라이브러리(패키지)를 설치하고 관리하는 도구이다. npm이 표준이지만, pnpm는 속도와 효율성 면에서 큰 이점이 있다. pnpm은 node_modules 폴더에 파일을 중복으로 설치하지 않고, 컴퓨터의 공용 저장소에서 링크를 가져오는 방식을 사용한다. 이 덕분에 설치 속도가 훨씬 빠르고 디스크 공간을 획기적으로 절약할 수 있다.

3. React 프로젝트 생성 및 디버깅 경험은 단순한 코드 작성이 아니라, 실제 '문제 해결'의 연속이었다. pnpm start가 아닌 **pnpm dev**를 써야 했던 이유는 package.json의 scripts가 프로젝트의 실행 명령을 정의하기 때문임을 배웠다. node_modules가 회색으로 보인 것은 오류가 아닌, .gitignore가 버전 관리에서 의도적으로 제외한 정상 상태임을 확인했다. ./pages/Home.jsx (파일 확장자 누락) 오류와 react-router-dom (라이브러리 설치 누락) 오류를 통해, React가 로컬 파일과 외부 패키지를 어떻게 인식하고 불러오는지 명확히 구분하게 되었다. 또한, 기능을 Navbar, Content, FooterButton으로 잘게 쪼개는 '컴포넌트' 기반 개발을 실습했다. 그리고 이 조각들을 Home.jsx에서 조립(import)해야 비로소 한 페이지가 완성된다는 것을 알게 되었다. onClick 기능이 화면이 아닌 브라우저 개발자 도구(F12)의 '콘솔(Console)'창에 출력된다는 점, 그리고 '장바구니 버튼 2개' 오류를 해결하며 컴포넌트 간 코드 중복을 제거하는 디버깅까지 직접 경험했다. 이 모든 과정은 오류를 만나고, 원인을 추론하며, 정확한 해결책을 적용하는, 실전적인 개발 그 자체였던 것 같다.
