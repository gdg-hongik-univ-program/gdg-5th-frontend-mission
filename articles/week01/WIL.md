## pnpm과 npm의 비교 및 선택
* 호이스팅기법으로 인한 유령 의존성이 발생하는 npm의 문제점을 이해했습니다. (`package.json`에 없는 패키지를 임의로 사용하여 나중에 오류 발생)
* 심볼릭 링크를 활용하여 디스크 공간을 절약하고, `node_modules` 구조를 엄격하게 관리하여 유령 의존성을 원천적으로 차단한다는 점을 학습했습니다.
* 장기적인 효율성과 엄격한 의존성 관리를 위해 `pnpm`을 주력 패키지 매니저로 선택하기로 했습니다.

## 개발 환경 구축 및 최적화

환경 설정의 중요성과 복잡성을 실감했으며 패키지 매니저의 작동 원리를 깊이 이해했습니다.

### 환경 설정 난관 및 해결 과정
* `pnpm exec`나 `npx` 명령어가 `Command not found` 오류를 반복적으로 발생시켜 TailwindCSS 설정 파일을 생성하지 못하는 문제을 겪었습니다.
* 한글이 문제인가 싶어서 `pnpm config set store-dir C:\pnpm-store` 명령어로 `pnpm`의 경로를 한글이 없는 경로로 강제 변경해보았지만 실패했습니다.
* 다시 깔아보는게 맞다고 생각해 `node_modules`와 `pnpm-lock.yaml`을 모두 삭제하고 `pnpm install`을 재실행해보았지만 실패했습니다.
* 결국엔 원인을 제미나이와 함께 계속 찾아갔으나 해결하지 못해 수동으로 파일을 직접 생성하면서 어떤 코드가 필요한지 학습했습니다.

## 쇼핑몰 UI 구현

Vite와 TailwindCSS를 사용하여 쇼핑몰 UI를 구현하며 React의 핵심 원리를 실습했습니다.

### 컴포넌트 구조 및 문법
* **컴포넌트 분리**: `Header`, `Content`, `Footer`로 분리하고, `App.jsx`에서 `<></>`로 하나의 최상위 요소로 묶어주는 JSX 규칙을 실습했습니다.
* **레이아웃이 겹치는 해결**:
    * 초기 코드에서 `Header`, `Content`, `Footer`에 모두 `fixed`를 사용하여 레이아웃이 겹치는 문제를 겪었습니다.
    * `App.jsx`의 `flex flex-col` 흐름을 유지하고, `Content.jsx`에 `flex-grow`를 적용하여 남은 공간을 자동으로 채우도록 수정했습니다.

### 기능 구현 및 State 관리
* **검색 기능**: `Content` 컴포넌트 내에서 `useState` 훅을 사용하여 `input` 태그의 값과 상태를 연결하는 제어 컴포넌트 패턴을 구현했습니다.
* **버튼 이벤트**: 모든 메뉴 및 버튼에 `onClick={() => console.log(...) }` 핸들러를 추가하여 미션 요구사항인 콘솔 출력을 완료했습니다.

### TailwindCSS 스타일링
* **높이 통일**: 검색창과 검색 버튼의 높이를 고정 높이를 주지 않고 두 요소의 수직 패딩(`py-12px`)을 동일하게 설정하여 높이를 일관성 있게 맞추었습니다.

## 1주차 미션 후기
* 이번 주차의 미션들을 수행하며 앞으로 프론트엔드를 구사하는데 문제 없도록 기본기를 제대로 다지는 내용들 내용들로 구성되었다는 것을 느꼈습니다.
* 그래서 내용들을 놓치지 않고 깊게 보고 싶다는 생각에 첫 번째 미션을 조금 디테일하게 했고 개념들을 위주로 머리 속에 잘 넣어가려고 했습니다.
* 이후 패키지 매니저의 차이를 학습하고 원하는 패키지 매니저를 선택하여 세번째 미션을 수행하려했는데 막상 하려니까 잘 되지 않았습니다.
* 같은 모습이 잘 구현되지 않았고 많은 시간이 소요되게 되어 기존 답안을 토대로 적절히 수정하는 방향으로 가기로 했습니다.
* 1주차는 미션 시간이 빠듯해서 아쉬웠고, 2주차부터는 시간을 더 할애하여 순전히 나의 생각으로만 미션을 수행해봐야겠다고 다짐했습니다.
